- Chains、Trees、Graphs 三種推理拓樸比較
    | 面向          | Chains（鏈式）      | Trees（樹狀）  | Graphs（圖狀）    |
    | ----------- | --------------- | ---------- | ------------- |
    | 核心結構        | 線性序列            | 分支樹狀       | 非線性圖狀         |
    | 基本概念        | Step-by-step 推理 | 多路徑探索      | 多節點互連與聚合      |
    | 推理方式        | 單一路徑            | 多分支搜尋      | 任意連結推理        |
    | 中介 thoughts | 有               | 有          | 有             |
    | 探索能力        | 低               | 高          | 非常高           |
    | 聚合能力        | 無               | 有限         | 強             |
    | 回溯能力        | 幾乎無             | 有          | 有             |
    | 多跳推理能力      | 普通              | 高          | 非常高           |
    | 任務複雜度適應性    | 低～中             | 中～高        | 高             |
    | 可解釋性        | 高               | 中          | 較低            |
    | 推理彈性        | 低               | 中          | 高             |
    | prompt 類型   | 多為單 prompt      | 多為多 prompt | 單／多 prompt 混合 |
    | 計算成本        | 低               | 中～高        | 高             |
    | token 消耗    | 少               | 多          | 中～多           |
    | 代表方法        | CoT             | ToT、AoT    | GoT、HoT       |
    | 最適合任務       | 線性推理            | 階層式問題      | 複雜互連問題        |
  - 成本與品質比較
    | 比較面向      | Chains | Trees | Graphs |
    | --------- | ------ | ----- | ------ |
    | 計算成本      | 最低     | 較高    | 最高     |
    | Prompt 數量 | 少      | 多     | 多      |
    | Token 使用量 | 少      | 多     | 中～多    |
    | 執行時間      | 快      | 較慢    | 最慢     |
    | 推理品質      | 中      | 高     | 非常高    |
    | 解空間探索能力   | 低      | 高     | 非常高    |
    | 適合複雜問題    | 較弱     | 強     | 非常強    |
    | 推理彈性      | 低      | 中     | 高      |
    | 成本效益      | 高      | 中     | 視任務而定  |
  - 不同任務類型之最佳推理拓樸
    | 任務類型  | 代表資料集／任務                 | 最佳方法    | 原因            |
    | ----- | ------------------------ | ------- | ------------- |
    | 算術推理  | GSM8K、SVAMP、DROP         | ToT、GoT | 可分解與回溯、降低錯誤傳播 |
    | 常識推理  | StrategyQA、MuSiQue       | GoT、ToT | 適合多跳知識整合      |
    | 符號推理  | Coin Flipping、SPARTUN    | ToT、GoT | 可系統化探索解空間     |
    | 特殊化推理 | HumanEval、MBPP、ScienceQA | GoT、ToT | 適合規劃與多步驟推理    |
    | 序列式推理 | 簡單數學、單跳 QA               | CoT、ToT | 線性流程效率高       |
  - 各推理拓樸優缺點比較
    | 拓樸     | 優點                   | 缺點                   |
    | ------ | -------------------- | -------------------- |
    | Chains | 成本低、可解釋性高、效率佳        | 易錯誤累積、缺乏探索能力         |
    | Trees  | 可探索多條路徑、支援回溯、提升準確率   | prompt 成本高、token 消耗大 |
    | Graphs | 支援非線性推理、聚合能力強、適合複雜任務 | 架構複雜、計算負擔大           |
  - Trees 與 Graphs 的進階能力比較
    | 能力               | Trees  | Graphs   |
    | ---------------- | ------ | -------- |
    | 分支探索             | 有      | 有        |
    | 聚合（Aggregation）  | 有限     | 強        |
    | 節點互動             | 父子節點為主 | 任意節點皆可互動 |
    | 回溯（Backtracking） | 有      | 有        |
    | 動態推理流程           | 中      | 高        |
    | 非階層式推理           | 不支援    | 支援       |
    | 多面向問題處理          | 中      | 高        |
    | 協同效應（Synergy）    | 有限     | 強        |
  - 評估技術（Evaluation Techniques）比較
    | 評估方法                              | 核心概念               | 目的                    |
    | --------------------------------- | ------------------ | --------------------- |
    | Final Answer Evaluation           | 評估最終答案正確性          | 檢查結果是否正確              |
    | Intermediate Reasoning Evaluation | 評估中介步驟             | 確保推理過程有效              |
    | Discriminative Evaluation         | 偵測邏輯不一致            | 驗證推理合理性               |
    | Generative Evaluation             | 生成可驗證推理步驟          | 測試結構化推理能力             |
    | OCEAN Framework                   | 使用 MDP + KG-IPS    | 評估 reasoning fidelity |
    | Robust Verifier                   | 驗證 self-correction | 提升推理可靠性               |
  - Chains、Trees、Graphs 發展方向比較
    | 面向    | Chains      | Trees        | Graphs       |
    | ----- | ----------- | ------------ | ------------ |
    | 目前成熟度 | 高           | 中            | 中            |
    | 架構複雜度 | 低           | 中            | 高            |
    | 研究重點  | 提升穩定性       | 降低搜尋成本       | 提升聚合效率       |
    | 未來方向  | 更佳 verifier | 單 prompt 樹編碼 | 單 prompt 圖編碼 |
    | 潛在優勢  | 高效率         | 高品質搜尋        | 高度泛化推理       |
    | 主要挑戰  | 錯誤累積        | 計算成本         | 拓樸管理複雜度      |
