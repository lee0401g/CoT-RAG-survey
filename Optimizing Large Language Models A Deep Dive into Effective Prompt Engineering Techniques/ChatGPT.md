# Optimizing Large Language Models: A Deep Dive into Effective Prompt Engineering Techniques

## 總結

> 這篇研究 探討大型語言模型在不同任務下如何透過提示工程優化效能，用 多種 Prompt Engineering 技術（ICL、CoT、RAG、SSR、ToT）與多模型多資料集實驗方法，發現 最佳提示策略取決於資料集與模型特性，且適當組合能顯著提升表現而無需重新訓練

### 盲區

> 閱讀盲區

- 各種 Prompt Engineering 技術（如 CoT、SSR、ToT）之間的細微差異不易直觀理解  
- 評估指標（BLEU、ROUGE、BERTScore 等）對不同任務的意義較抽象  
- 不同資料集（推理、語言理解、數學）如何影響模型表現需要背景知識  
- 為何同一方法在不同模型上效果不同，缺乏直觀解釋  

> 本研究盲區

- 僅測試特定三種模型（Gemma2、Llama3、Mistral），泛化性有限  
- Prompt 設計細節與模板未完全標準化，可能影響公平比較  
- 未深入分析計算成本與延遲（實務部署關鍵）  
- 缺乏人類主觀評估（僅依賴自動指標）  
- 未涵蓋更多新興方法（如 agent-based prompting）

## IMRaD

> 問題是什麼?為何重要?研究動機與背景

- 大型語言模型雖強大，但容易產生錯誤或不符合意圖的回答  
- 模型微調成本高，因此需要低成本優化方法  
- Prompt Engineering 可在推論階段改善輸出品質  
- 不同任務（推理、語言理解、數學）對提示策略需求不同  
- 缺乏系統性比較各種 Prompt 技術在不同場景下的效果  

> 他們怎麼做?用了哪些數據、模型與技術

- 使用三種主流模型：Gemma2、Llama3、Mistral  
- 採用多種 Prompt Engineering 技術：  
  - ICL（情境學習）  
  - CoT（思維鏈）  
  - RAG（檢索增強生成）  
  - SSR（逐步推理）  
  - ToT（思維樹）  
- 在六個資料集上測試：  
  - ARC（科學推理）  
  - HellaSwag（常識理解）  
  - MMLU（多領域知識）  
  - TruthfulQA（真實性）  
  - Winogrande（常識推理）  
  - GSM8K（數學推理）  
- 使用多種評估指標：BLEU、ROUGE、METEOR、BLEURT、BERTScore  
- 測試不同 Prompt 組合（單一與混合策略）  

> 做出什麼成果?圖表顯示了什麼發現?

- 不同任務適合不同 Prompt 技術：  
  - 數學/推理任務：CoT、SSR、ToT 表現最佳  
  - 語言理解任務：ICL 效果較佳  
  - 事實性任務：RAG 提升最明顯  
- Prompt 組合通常優於單一方法  
- 最佳策略依賴模型與資料集（無通用最佳解）  
- 高階模型對 Prompt 依賴降低，但提升幅度更大  
- RAG + Prompt 前處理效果優於單純 RAG  
- Gemma2 在多數情況表現最佳，但需搭配適當 Prompt  

> 結果代表什麼?有那些限制與未來延伸?

- Prompt Engineering 是低成本高效能優化手段  
- 未來應針對「模型 × 任務」自動選擇最佳 Prompt  
- 顯示 AI 系統需走向「策略自適應」而非固定設計  
- 限制包含模型範圍有限與評估方式單一  
- 未來可研究：  
  - 自動 Prompt 搜尋與生成  
  - 與微調方法的混合策略  
  - 即時系統中的效能與成本平衡

## 其他視角

> 審稿人角度

1. 是否存在 Prompt 設計偏差導致結果不公平？  
2. 不同模型是否使用完全一致的輸入格式與參數？  
3. 評估指標是否能真實反映使用者滿意度？  
4. 是否考慮計算成本與實務部署限制？  
5. 結論是否可泛化到其他未測試模型？  

---

> 如果應用到教育領域，需要哪些修改

- 設計適合學生程度的 Prompt（簡化語言與步驟）  
- 強化解釋性輸出（偏向 CoT / SSR）  
- 加入錯誤分析與回饋機制  
- 結合教材內容進行 RAG（課本、講義）  
- 控制生成內容避免誤導學生

> 教學式解釋（給初學者）

- 大型語言模型就像一個很聰明但有時會亂猜的學生  
- Prompt 就是你「怎麼問問題」  
- 不同問法會讓答案差很多，例如：  
  - 直接問 → 普通回答  
  - 要求一步一步解 → 更正確（CoT）  
  - 給例子 → 更容易學會（ICL）  
  - 提供資料 → 更準確（RAG）  
- 這篇研究就是在找：  
  「什麼問法在什麼情況下最好？」  

---

初學者最容易誤解的地方：

- 誤以為 Prompt Engineering 可以取代模型訓練（其實是補充）  
- 認為某一種方法（如 CoT）在所有任務都最好  
- 忽略資料集特性對結果的影響  
- 把評估指標當成絕對真實品質  
- 低估 Prompt 組合策略的重要性
